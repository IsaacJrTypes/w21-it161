Welcome

I have implemented the concepts I learned in WEB 161 to create 8 responsive web pages. Here is a quick overview of each page with a rundown of the skills I displayed.

  1. Accessibility
    1. A brief write-up on the importance of keeping in mind of accessibility when designing a website
    2. Use media queries to create a responsive web page and navigation bar.
    3. Citation aside bar contains an order list that drops down using media queries
    4. Responsive embedded YouTube video of a presentation on accessibility fundamentals
  2. Social Media Optimization
    1. A brief write-up on the importance of SMO to increase traffic to your site.
    2. Use title attribute to display title of the image presented and used alt attribute for accessibility.
    3. Responsive embedded YouTube video of a presentation on 20 ways to optimize your social media posts.
  3. Forms
    1. A brief overview of how forms have changed how users interact with websites.
    2. Used pre and code tags to display pieces of code that can change the way users can have a better form experience.
    3. Use media queries and display declarations to switch resolution of images displayed depending on userâ€™s screen size.
    4. Used divs tags with class and id attributes so images stay centered within section block.
  4. Gallery
    1. Used inline block value to create a row of images that respond to shrinking page size
    2. Implemented lightbox 2 to create a gallery upon click.
    3. Used CSS to center and uppercase description text under the images.
    4. Included resources to create a gallery of your own.
  5. Youtube
    1. Used iframe to include 2 responsive web streams.
  6. Calender
    1. Use media queries and display declarations to switch calendar view between agenda and month view.
    2. Included an upcoming section to include concert event details with the use of br tag to format text.
    3. Used I frame to include a Spotify playlist related to the event so users can quickly sample the artist performing at the event.
  7. Maps
    1. Used iframe to embed a responsive Google Map.
    2. Applied address tags to contact aside section with a href link, allowing mobile users to call the phone number by clicking the link.
    3. Used z‑index to prevent google + drop‑down menu from clipping under the map.
  8. Contact
    1. Used divs to wrap sections of the contact areas and used CSS to create spacing between contact form areas.
    2. Included a compact version of reCAPTCHA

I enjoyed developing this site, even though at times development became frustrating. Building this site has motivated me to practice building websites using front‑end mentor.io to help me make programming mistakes. By creating more mistakes in my work, I will learn from them and cut down development time. I think I must develop over 100 projects to become comfortable with programming websites. Since I am in the early stages of learning web development, quantity over quality is my priority as each project I develop, the quality will improve.
